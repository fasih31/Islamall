🛠️ 2. Database Design for Knowledge Retrieval

Database tables/entities (high-level)

2.1 Books Table

Books (
  id SERIAL PRIMARY KEY,
  title VARCHAR,
  author VARCHAR,
  category VARCHAR,        -- e.g., Finance, Politics, Culture
  language VARCHAR,        -- Arabic, Urdu, English
  description TEXT,
  source_url TEXT,
  pdf_url TEXT,
  published_year INT
);

2.2 Topics Table

Topics (
  id SERIAL PRIMARY KEY,
  name VARCHAR,            -- e.g., Riba, Salah, Fasting
  section VARCHAR,         -- Finance, Daily Life, etc.
  summary TEXT
);

2.3 TopicContent Table

TopicContent (
  id SERIAL PRIMARY KEY,
  topic_id INT REFERENCES Topics(id),
  book_id INT REFERENCES Books(id),
  quote TEXT,             -- Excerpt or explanation
  page_number VARCHAR,
  reference_type VARCHAR,  -- Quran, Hadith, Book
  reference_id VARCHAR,    -- e.g., Surah:Ayah, Hadith Number, Page Number
  authenticity_grade VARCHAR,  -- For Hadith: Sahih, Hasan, Daif
  language VARCHAR
);

2.4 Quran Table

Quran (
  id SERIAL PRIMARY KEY,
  surah INT,
  ayah INT,
  text_ar TEXT,
  translations JSONB,  -- { 'en': '...', 'ur': '...', 'hi': '...'}
  tafsir JSONB         -- list of tafsir references
);

2.5 Hadith Table

Hadith (
  id SERIAL PRIMARY KEY,
  collection VARCHAR,
  book VARCHAR,
  chapter VARCHAR,
  hadith_number INT,
  text_ar TEXT,
  translation JSONB,       -- { 'en': '...', 'ur': '...' }
  isnad TEXT,
  grade VARCHAR,           -- Sahih, Hasan, Daif
  book_reference_id INT
);

🧠 3. Functionality / Features for Each Section

A. Topic Exploration
	•	Dynamic topic list for each section (Finance → Riba, Zakat, Contracts)

	1.	Summary definition

C. AI-Powered Explanation (Optional Advanced)

	•	Links to full reading in books or PDFs

D. Multi-Language Support
	•	Arabic, Urdu, English, Hindi for all sections
	•	User can toggle language per topic, Quran verse, Hadith, or book
